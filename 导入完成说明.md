## 🚀 快速开始（3 步）

### 步骤 1：获取 Supabase Service Role Key

1. 登录 [Supabase Dashboard](https://supabase.com/dashboard)
2. 选择项目 **showcase-website**
3. 进入 **Settings** → **API**
4. 复制 **service_role** key（⚠️ 请妥善保管）

### 步骤 2：运行导入脚本

**方式 A：一键导入（推荐）**

直接双击运行：`run-import.bat`

脚本会提示你输入 Supabase Service Role Key，然后自动导入所有案例数据。

**方式 B：命令行导入**

```bash
# 设置环境变量
set SUPABASE_SERVICE_ROLE_KEY=your_actual_service_role_key

# 运行导入
node simple-import.js
```

### 步骤 3：访问后台

1. 打开浏览器：`http://localhost:3000`
2. 登录管理员账号
3. 进入管理后台查看案例列表

|------|------|

## 🐛 常见问题

### Q1: 提示"未找到 Service Role Key"

**A**: 按照步骤 1 获取 Service Role Key，然后运行 `run-import.bat`。

### Q2: 提示"数据表不存在"

**A**: 需要先在 Supabase SQL Editor 中创建 `cases` 表。参考 `migrations/001_create_cases_table.sql`。

### Q3: 图片没有导入

**A**:
- 如果使用 `simple-import.js`，只导入案例数据，不导入图片
- 如果需要导入图片，使用 `full-import.js`
- 检查 `src/public/images/cases/` 目录是否存在图片文件

### Q4: 导入失败，提示权限错误

**A**:
- 检查 Service Role Key 是否正确
- 检查 Supabase 项目是否已启动
- 检查网络连接

### Q5: 想重新导入

**A**: 脚本会自动跳过已存在的案例（根据标题判断）。如需完全重新导入，手动删除数据库中的记录。
