# 🚀 手动导入（最简单的方法）

## 方法 1：直接输入命令（推荐）

### 步骤：

1. **打开命令提示符（CMD）**

2. **输入以下命令**（复制粘贴即可）：

```cmd
cd /d C:\Users\zyq15\.openclaw\workspace\architecture-showcase
```

3. **设置环境变量**（替换下面的 key 为你的实际 key）：

```cmd
set SUPABASE_SERVICE_ROLE_KEY=你的service_role_key
```

4. **运行导入**：

```cmd
node simple-import.js
```

---

## 方法 2：使用批处理文件（不闪退）

1. **右键点击**：`导入.bat`
2. 选择 **"以管理员身份运行"**
3. 输入 Service Role Key
4. 等待完成

---

## 方法 3：使用 PowerShell 脚本

1. **右键点击**：`import-stable.ps1`
2. 选择 **"使用 PowerShell 运行"**
3. 输入 Service Role Key
4. 等待完成

---

## 📋 Service Role Key 获取方法

1. 访问 https://supabase.com/dashboard
2. 选择项目 **showcase-website**
3. 点击左侧 **Settings** → **API**
4. 在 **Project API keys** 部分，找到 **service_role** key
5. **复制**这个 key（以 `eyJ` 开头，很长的一串字符）

⚠️ **注意**：
- 不要用 anon key，要用 service_role key
- key 以 `eyJ` 开头
- 长度大约 50-60 个字符

## ❓ 遇到问题？

### 问题 1：提示"未找到 simple-import.js"

**解决**：
```cmd
cd /d C:\Users\zyq15\.openclaw\workspace\architecture-showcase
```
确保你在项目目录下。

### 问题 2：提示"未找到 Node.js"

**解决**：
1. 下载 Node.js: https://nodejs.org
2. 安装后重启命令行
3. 运行 `node --version` 确认安装成功

### 问题 3：提示"导入失败"

**解决**：
1. 检查 Service Role Key 是否正确
2. 检查 Supabase 项目是否已启动
3. 检查数据库表是否已创建（cases 表）
4. 检查网络连接
